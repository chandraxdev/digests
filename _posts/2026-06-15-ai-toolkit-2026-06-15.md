New AI assets from the past week — freshly shipped MCP servers, skills and plugins for Claude, ChatGPT and Gemini. These are newer and smaller than the established staples, and that's the point: this issue tracks what just landed, with real ship dates, not an all-time list.

## New MCP servers & tools

**[ocpp-rag — an MCP server that knows the EV-charging specs](https://github.com/nader0913/ocpp-rag)**  
A Model Context Protocol server with a built-in RAG knowledge base for the OCPP charging protocols — a tidy example of packaging deep domain docs behind an assistant. Picked up ~110 stars in days.  
*GitHub · created Jun 9*

**[agent-harness-generator — scaffold your own agent harness](https://github.com/ruvnet/agent-harness-generator)**  
From the author of ruflo: a generator that scaffolds a focused, single-purpose harness for an AI agent rather than reaching for a heavy framework.  
*GitHub · created Jun 13*

**[A self-hosted MCP server registry and proxy](https://dev.to/diya730/i-built-a-self-hosted-mcp-server-registry-and-proxy-in-typescript-12b)**  
A write-up plus code for running your own registry/proxy in front of multiple MCP servers — useful once you've installed more than a couple.  
*dev.to · Jun 13*

**[Dusk — Playwright-style MCP, but for Flutter apps](https://dev.to/fluttersdk/i-built-dusk-playwright-mcp-but-for-flutter-apps-l5m)**  
Brings browser-automation-style control to Flutter app testing through MCP — handy if your agent needs to drive a Flutter UI.  
*dev.to · Jun 14*

## New skills & plugins

**[designer-skill — give a coding agent a real design voice](https://github.com/Pythoughts-labs/designer-skill)**  
An MCP server/skill that hands coding agents production-grade design sensibility, so generated UIs look less generic.  
*GitHub · created Jun 9*

**[skillreaper — prune your agent-skill stack](https://github.com/thousandflowers/skillreaper)**  
A scan-report-prune tool for the skills you've accumulated: evidence-based cleanup so an agent isn't carrying dead weight.  
*GitHub · created Jun 10*

**[A Claude Code plugin for MV3 Chrome extensions](https://dev.to/harish_kumar/what-i-learned-building-a-claude-code-plugin-for-mv3-chrome-extensions-514i)**  
Field notes (and a plugin) from building a Claude Code plugin targeting Manifest V3 Chrome extensions — a concrete look at extending Claude Code itself.  
*dev.to · Jun 13*

**[A context-window packer for Claude](https://github.com/Supereldecoder/claude-opus48-context-packer)**  
A high-density context/token optimiser aimed at Claude — squeezes more useful material into the window. New and small, but a sharp idea.  
*GitHub · created Jun 13*
