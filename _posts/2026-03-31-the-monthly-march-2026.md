March belonged to the coding agent: an open-source contender in OpenCode, smarter and cheaper ways to run inference for them, and a steady run of essays on how to reshape software — CLIs, workflows, even programming languages — around agents. Elsewhere, a rare clean-slate routing story out of Switzerland, a couple of notable open-weights model drops, and a meta-prompting system that caught fire.

## AI Engineering

**[Quantization from the ground up](https://ngrok.com/blog/quantization)**  
A clear, first-principles walk through model quantization — the rare explainer that actually builds the intuition rather than reciting the acronyms.  
*ngrok · Mar 25*

**[You need to rewrite your CLI for AI agents](https://justin.poehnelt.com/posts/rewrite-your-cli-for-ai-agents/)**  
A persuasive argument that command-line tools designed for humans make poor tools for agents, and what to change.  
*justin.poehnelt.com · Mar 4*

**[Parallel coding agents with tmux and Markdown specs](https://schipper.ai/posts/parallel-coding-agents/)**  
A practical workflow for running several coding agents at once, coordinated through plain Markdown specifications.  
*schipper.ai · Mar 2*

**[Hypura: a storage-tier-aware inference scheduler for Apple Silicon](https://github.com/t8/hypura)**  
A neat systems idea — schedule LLM inference with awareness of the memory/storage hierarchy on Apple hardware.  
*GitHub · Mar 24*

**[Coding agents could make free software matter again](https://www.gjlondon.com/blog/ai-agents-could-make-free-software-matter-again/)**  
A thoughtful essay on how cheap, capable agents might shift the economics back toward open source.  
*gjlondon.com · Mar 29*

## Applied AI Tips

**[Get Shit Done: a meta-prompting, context-engineering, spec-driven dev system](https://github.com/gsd-build/get-shit-done)**  
A complete, opinionated system for driving agents through specs and structured context rather than ad-hoc prompting — one of the month's most-shared.  
*GitHub · Mar 17*

**[Prompt engineering for humans](https://michaelheap.com/prompt-engineering-for-humans/)**  
A grounded, jargon-free take on prompting that focuses on clear thinking over tricks.  
*michaelheap.com · Mar 31*

**[An open security database for agent skills](https://index.tego.security/skills/)**  
An early attempt to catalogue the security properties of agent skills — useful as the skills format spreads.  
*tego.security · Mar 16*
